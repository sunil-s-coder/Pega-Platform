# Application Development Studios:

## App Studio vs Dev Studio (Pega Platform)

| **Feature / Focus** | **App Studio** | **Dev Studio** |
|----------------------|----------------|----------------|
| **Types of App Developers** | Domain Experts:<br>• Business Analysts<br>• Citizen Developers<br>• Front-end Developers | Implementation Experts:<br>• System Architects<br>• Full-stack Developers<br>• Database Administrators<br>• Security Administrators |
| **Primary Purpose** | Simplify core application development | Enable advanced and complex configuration |
| **Key Capabilities** | • Case design<br>• Data management<br>• User experience design | • Direct access to Rule forms<br>• Advanced Rule configuration |
| **Additional Features** | Focus on improving development outcomes through business knowledge | • Configure security permissions & access control<br>• Manage Rules for reuse<br>• Address performance limitations |
| **Complexity Level** | Beginner to Intermediate (business-friendly) | Advanced (technical, detailed control) |

---

## Characteristics of the Pega Platform

- **Co-develop Solutions:**  
  Domain experts and implementation experts can collaborate to design and build better solutions.

- **Reusability:**  
  - In **Dev Studio**, developers create standardized and reusable Rules.  
  - In **App Studio**, domain developers can leverage these Rules to accelerate development and ensure consistency across applications.

- **Switch Between Studios:**  
  Implementation developers can switch between **App Studio** and **Dev Studio** as needed to configure and refine Rule behavior.

- **Naming Differences Between Studios:**

  | **App Studio**       | **Dev Studio**                   |
  |----------------------|----------------------------------|
  | Field                | Property                         |
  | Goal and Deadline    | Service-Level Agreement (SLA)     |
  | User                 | Operator                         |
  | Team                 | Work Group                       |
